## 七、数据流设计

### 7.1 数据存储架构

```
Obsidian Vault
├── .obsidian/
│   └── plugins/
│       └── obsidian-database/
│           ├── main.js
│           ├── manifest.json
│           └── styles.css
├── databases/
│   ├── projects.database.md
│   ├── reading-list.database.md
│   └── tasks.database.md
└── notes/
    ├── daily/
    │   └── 2026-06-06.md
    └── projects/
        └── project-a.md
```

### 7.2 数据格式示例

```markdown
---
type: database
version: 1
created: 2026-06-06T00:00:00Z
updated: 2026-06-06T12:00:00Z
aliases: ["项目管理", "任务看板"]
properties:
  - id: prop_title
    name: "任务名称"
    type: text
  - id: prop_status
    name: "状态"
    type: select
    options:
      - {id: opt_todo, name: "待办", color: "gray"}
      - {id: opt_doing, name: "进行中", color: "blue"}
      - {id: opt_done, name: "完成", color: "green"}
  - id: prop_date
    name: "截止日期"
    type: date
views:
  - id: view_table
    name: "表格视图"
    type: table
rows:
  - id: row_1
    cells:
      prop_title: "完成插件设计"
      prop_status: opt_doing
      prop_date: "2026-06-15"
  - id: row_2
    cells:
      prop_title: "编写文档"
      prop_status: opt_todo
      prop_date: "2026-06-20"
---

# 项目管理

这是一个用于管理项目任务的数据库。
```

### 7.3 数据流向

**读取数据流**：
```
用户打开数据库 → 代码块处理器触发 → 读取 frontmatter 元数据 → 解析属性定义 → 解析行数据 → 应用筛选/排序 → 渲染视图 → 显示给用户
```

**写入数据流**：
```
用户编辑单元格 → 触发数据验证 → 更新内存数据 → 序列化为 frontmatter → 写入文件 → 触发事件通知 → 更新视图
```

**链接数据流**：
```
用户输入 [[链接]] → 解析链接语法 → 查找目标数据库 → 渲染链接/嵌入视图 → 用户点击链接 → 打开目标数据库
```

## 八、用户使用流程推演

### 8.1 新用户首次使用流程

1. 安装插件（从 Obsidian 社区插件库安装，启用插件）
2. 创建第一个数据库（Ctrl+P → "Create Database" → 输入名称 → 选择模板）
3. 配置属性（添加列 → 选择属性类型 → 输入属性名称）
4. 添加数据（点击 "添加行" → 双击单元格编辑 → 选择属性值）
5. 使用视图（查看表格视图 → 筛选和排序数据）
6. 链接和嵌入（在笔记中输入 [[数据库名]] → 使用 ![[数据库名]] 嵌入视图）

### 8.2 日常使用流程

1. 打开数据库（从文件列表/链接跳转/嵌入视图打开）
2. 查看数据（浏览表格 → 使用筛选查找 → 使用排序整理）
3. 编辑数据（双击单元格编辑 → 拖拽行排序 → 批量操作多行）
4. 管理数据库（添加/删除列 → 调整列宽 → 修改属性配置）
5. 链接笔记（在笔记中引用数据库 → 在数据库中链接笔记 → 查看反向链接）

### 8.3 高级使用流程

1. 多视图切换（表格 → 看板 → 日历，后续版本）
2. 关联数据（创建关联属性 → 选择关联行 → 查看汇总数据，后续版本）
3. 公式计算（创建公式属性 → 输入公式表达式 → 查看计算结果，后续版本）
4. 数据导入导出（从 Markdown 表格导入 → 导出为 CSV/Excel → 从 Notion 导入）
5. 模板管理（使用预设模板 → 创建自定义模板 → 分享模板到社区）

## 九、Obsidian 编辑语法集成规范

### 9.1 Markdown 基础语法支持

| 语法 | 说明 | 示例 |
|------|------|------|
| `**粗体**` | 单元格内粗体文本 | `**重要任务**` |
| `*斜体*` | 单元格内斜体文本 | `*备注内容*` |
| `~~删除线~~` | 单元格内删除线 | `~~已完成~~` |
| `` `代码` `` | 单元格内行内代码 | `` `function()` `` |
| `[链接](url)` | 标准 Markdown 链接 | `[Google](https://google.com)` |
| `![](image.png)` | 单元格内图片 | `![](screenshot.png)` |
| `==高亮==` | Obsidian 高亮语法 | `==重点内容==` |
| `^上标^` | Obsidian 上标语法 | `x^2^` |
