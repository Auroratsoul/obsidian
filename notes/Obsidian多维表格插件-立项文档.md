## 一、项目立项说明

### 1.1 产品定义

**产品名称**：Obsidian Database（暂定）

**产品定位**：Obsidian 生态中的开放平台型多维表格插件，提供结构化数据管理能力。

**产品形态**：Obsidian 社区插件，使用 TypeScript 开发，数据存储在本地 vault 的 Markdown 文件中。

**核心价值**：将 Notion 的多维表格能力带入 Obsidian，同时保持本地优先、隐私安全的特性。

### 1.2 目标用户

| 用户画像 | 说明 |
|----------|------|
| **主要用户** | Obsidian 深度用户，需要在笔记中管理结构化数据 |
| **次要用户** | 从 Notion 迁移的用户，希望本地化管理数据 |
| **潜在用户** | 项目管理、知识管理、任务管理需求者 |

### 1.3 解决问题

| 问题 | 现状 | 解决方案 |
|------|------|----------|
| 结构化数据管理 | Obsidian 只有纯文本和 Markdown 表格 | 提供多维表格视图 |
| 可视化数据展示 | 缺少看板、日历等可视化视图 | 提供多种视图类型 |
| 知识关联 | 笔记间关联能力有限 | 支持关联属性和双向链接 |
| 任务管理 | 缺少任务跟踪工具 | 提供状态、日期、优先级等属性 |
| 项目管理 | 缺少时间线管理 | 提供甘特图视图 |

### 1.4 项目价值

| 价值维度 | 说明 |
|----------|------|
| **用户价值** | 在 Obsidian 中获得 Notion 级别的数据管理能力 |
| **生态价值** | 补充 Obsidian 生态的结构化数据管理空白 |
| **技术价值** | 建立可扩展的插件架构，支持未来功能扩展 |
| **社区价值** | 开源项目，促进 Obsidian 社区发展 |

### 1.5 第一版核心目标

**MVP 目标**：发布一个可用的表格视图插件，让用户能够在 Obsidian 中创建和管理结构化数据。

**时间目标**：1 个月内完成 MVP 开发并发布到 Obsidian 社区插件库。

**质量目标**：
- 插件可正常加载和卸载
- 表格视图可正常显示和编辑数据
- 支持 5 种基础属性类型
- UI 风格与 Notion 一致
- 支持亮色和暗色主题

### 2.3 暂放弃功能

| 功能类别 | 放弃功能 | 原因 |
|----------|----------|------|
| 实时协作 | 多人实时编辑、冲突合并、操作同步 | Obsidian 是本地优先工具 |
| 云端功能 | 云端存储、用户账户、数据同步、API 服务 | 数据必须存储在本地 vault |
| 复杂计算 | 电子表格公式、复杂数据分析、图表生成、数据透视表 | 超出笔记工具范畴 |
| 完整 Notion | Notion API、AI 功能、Web Clipper、表单视图 | 不是 Notion 客户端 |
| 高级 UI | 富文本编辑器、代码编辑器、图片/视频/PDF 编辑器 | 超出范围 |
| 安全功能 | 数据加密、访问控制、审计日志、自定义脚本 | 安全风险高或依赖外部 |
| 移动端 | 移动端完美体验、手势操作 | 移动端 API 限制多 |

## 三、产品流程与用户路径

### 3.1 新用户首次使用流程

```
1. 安装插件
   - 从 Obsidian 社区插件库安装
   - 启用插件

2. 创建第一个数据库
   - 打开命令面板 (Ctrl+P)
   - 输入 "Create Database"
   - 输入数据库名称
   - 选择模板（或空白）

3. 配置属性
   - 添加列（点击 "+" 按钮）
   - 选择属性类型
   - 输入属性名称

4. 添加数据
   - 点击 "添加行" 按钮
   - 双击单元格编辑
   - 选择属性值

5. 使用视图
   - 查看表格视图
   - 筛选和排序数据

6. 链接和嵌入
   - 在笔记中输入 [[数据库名]]
   - 使用 ![[数据库名]] 嵌入视图
```

### 3.2 日常使用流程

```
1. 打开数据库
   - 从文件列表打开
   - 从链接跳转打开
   - 从嵌入视图打开

2. 查看数据
   - 浏览表格
   - 使用筛选查找数据
   - 使用排序整理数据

3. 编辑数据
   - 双击单元格编辑
   - 拖拽行排序
   - 批量操作多行

4. 管理数据库
   - 添加/删除列
   - 调整列宽
   - 修改属性配置

5. 链接笔记
   - 在笔记中引用数据库
   - 在数据库中链接笔记
   - 查看反向链接
```

### 3.3 数据流转路径

#### 3.3.1 读取数据流
```
用户打开数据库
    ↓
代码块处理器触发
    ↓
读取 frontmatter 元数据
    ↓
解析属性定义
    ↓
解析行数据
    ↓
应用筛选/排序
    ↓
渲染视图
    ↓
显示给用户
```

#### 3.3.2 写入数据流
```
用户编辑单元格
    ↓
触发数据验证
    ↓
更新内存数据
    ↓
序列化为 frontmatter
    ↓
写入文件
    ↓
触发事件通知
    ↓
更新视图
```

#### 3.3.3 链接数据流
```
用户输入 [[链接]]
    ↓
解析链接语法
    ↓
查找目标数据库
    ↓
渲染链接/嵌入视图
    ↓
用户点击链接
    ↓
打开目标数据库
```

## 四、数据和业务对象

### 4.1 核心实体

#### 4.1.1 数据库（Database）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| name | string | 数据库名称 |
| type | string | 固定为 "database" |
| version | number | 数据格式版本 |
| created | Date | 创建时间 |
| updated | Date | 更新时间 |
| aliases | string[] | 别名列表 |
| properties | Property[] | 属性定义列表 |
| views | View[] | 视图配置列表 |
| rows | Row[] | 行数据列表 |

#### 4.1.2 属性（Property）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| name | string | 属性名称 |
| type | string | 属性类型 |
| options | Option[] | 选项配置（Select 类型） |
| formula | string | 公式表达式（Formula 类型） |
| relation | object | 关联配置（Relation 类型） |
| rollup | object | 汇总配置（Rollup 类型） |
| validation | object | 验证规则 |
| display | object | 显示配置 |

#### 4.1.3 行（Row）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| cells | object | 单元格数据（key 为属性 id） |

#### 4.1.4 视图（View）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| name | string | 视图名称 |
| type | string | 视图类型（table、board、calendar、gantt） |
| config | object | 视图配置（筛选、排序、分组等） |

#### 4.1.5 选项（Option）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 唯一标识 |
| name | string | 选项名称 |
| color | string | 颜色标识 |

### 4.2 实体关系

```
Database (1) ──→ (N) Property
Database (1) ──→ (N) View
Database (1) ──→ (N) Row
Property  (1) ──→ (N) Option (Select 类型)
Row       (1) ──→ (N) Cell (通过 cells 对象)
```

### 4.3 数据格式示例

```markdown
---
type: database
version: 1
created: 2026-06-06T00:00:00Z
updated: 2026-06-06T12:00:00Z
aliases: ["项目管理", "任务看板"]
properties:
  - id: prop_title
    name: "任务名称"
    type: text
  - id: prop_status
    name: "状态"
    type: select
    options:
      - {id: opt_todo, name: "待办", color: "gray"}
      - {id: opt_doing, name: "进行中", color: "blue"}
      - {id: opt_done, name: "完成", color: "green"}
  - id: prop_date
    name: "截止日期"
    type: date
views:
  - id: view_table
    name: "表格视图"
    type: table
rows:
  - id: row_1
    cells:
      prop_title: "完成插件设计"
      prop_status: opt_doing
      prop_date: "2026-06-15"
  - id: row_2
    cells:
      prop_title: "编写文档"
      prop_status: opt_todo
      prop_date: "2026-06-20"
---

# 项目管理

这是一个用于管理项目任务的数据库。
```

## 六、项目爆点和差异点

### 6.1 竞品分析

| 竞品 | 定位 | 优势 | 劣势 |
|------|------|------|------|
| **Dataview** | 查询引擎 | 强大查询能力 | 学习曲线陡峭、无可视化编辑 |
| **DB Folder** | 文件夹数据库 | 与文件系统集成 | UI 粗糙、性能一般 |
| **Projects** | 项目管理 | 多视图支持 | 功能较简单 |
| **Notion** | 全能工作空间 | 功能完整、UI 精美 | 云端依赖、隐私顾虑 |
| **Airtable** | 在线数据库 | 功能强大、API 丰富 | 价格贵、云端依赖 |

### 6.2 核心差异点

| 差异点 | 说明 | 竞争优势 |
|--------|------|----------|
| **本地优先** | 数据存储在 Obsidian vault 中 | 数据隐私、离线可用、与笔记集成 |
| **Obsidian 深度集成** | 支持 `[[]]`、`![[ ]]`、Markdown 语法 | 无缝融入 Obsidian 生态 |
| **Notion 风格 UI** | 复刻 Notion 的视觉和交互 | 熟悉的用户体验 |
| **开放平台架构** | 可扩展，支持未来插件生态 | 长期发展潜力 |
| **免费开源** | 社区驱动，无费用 | 降低使用门槛 |

### 6.3 用户吸引力

| 吸引点 | 目标用户 | 价值主张 |
|--------|----------|----------|
| Notion 体验本地化 | 从 Notion 迁移的用户 | 保留 Notion 体验，数据更安全 |
| 结构化数据管理 | Obsidian 深度用户 | 在笔记中管理结构化数据 |
| 多视图可视化 | 项目管理需求者 | 表格、看板、日历等多种视图 |
| 知识关联能力 | 知识管理需求者 | 建立笔记间的关联关系 |
| 免费开源 | 预算敏感用户 | 无费用，社区支持 |
